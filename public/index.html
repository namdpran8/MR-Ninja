The Problem

GitLab Duo Agent Platform caps context at ~200,000 tokens per call

⚠️

Truncated Reviews

Large MRs get cut off mid-analysis, missing critical vulnerabilities in later files

🔓

Incomplete Scans

Security checks skip files when context limit is hit, creating compliance gaps

😤

Developer Frustration

Teams can't get automated analysis for their monorepo MRs with 500+ changed files

📊

Enterprise Scale

A single monorepo MR can generate 500k-1M tokens of diff content

The Solution

Intelligent chunking, specialist processing, and unified aggregation

🎯

Smart Prioritization

Files are classified into 6 priority tiers (P1-P6). Security-critical files like .env, Dockerfile, and auth handlers are analyzed first.

🧩

Intelligent Chunking

Bin-packing algorithm creates optimally-sized chunks (~70k tokens each) while respecting file boundaries and priorities.

🤖

Specialist Agents

Each chunk is routed to the right agent: Security Analyst for P1 files, Code Reviewer for logic, Dependency Analyzer for manifests.

🔗

Context Carryover

Critical findings and open questions persist across chunks via compact summaries, ensuring continuity.

📊

Unified Aggregation

Findings are deduplicated, ranked by severity, and synthesized into a single report with risk scoring.

💬

GitLab Native

Posts analysis directly as MR comments, integrates with CI/CD, and uses the GitLab API throughout.

How It Works

Five-stage pipeline from detection to reporting

1

Detect

Estimate token footprint using len(text) / 4 heuristic. If >150k tokens, activate chunking.

→
2

Plan

Classify files by priority (security-critical first, tests last). Bin-pack into ~70k-token chunks.

→
3

Process

Run each chunk through specialist agents: Security Analyst, Code Reviewer, Dependency Analyzer.

→
4

Carry Context

Generate compact cross-chunk summaries. Critical/High findings and open questions persist.

→
5

Aggregate

Deduplicate findings, rank by severity, calculate risk score, generate unified Markdown report.

See It In Action

Run a simulated 512-file MR analysis with no GitLab account required

# Install dependencies
pip install -r requirements.txt

# Run demo simulation (512 files)
python -m demo.simulate_large_mr

# Custom file count
python -m demo.simulate_large_mr --files 1000

# Save report to file
python -m demo.simulate_large_mr --files 512 --output report.md

Quick Start

Get Mr Ninja running in under 2 minutes

1

Clone the Repo

git clone https://gitlab.com/your-group/mr-ninja.git
cd mr-ninja
2

Install Dependencies

pip install -r requirements.txt
3

Run the Demo

python -m demo.simulate_large_mr

Or use Docker

docker build -t mr-ninja .
docker run -p 8000:8000 -e GITLAB_TOKEN=glpat-xxx mr-ninja
